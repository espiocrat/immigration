# 출입국 행정 투명성 DX 대시보드

> 공공데이터(법무부 통계월보)를 가공해 **출입국 사무소·비자 종류별 비자 처리 예상기간**을 예측하고, 인터랙티브 대시보드로 공개하는 디지털 전환(DX) 프로젝트입니다.

한국에 체류하는 약 287만 외국인은 비자 변경·연장 신청 후 "언제 처리되는지" 알 수 없는 이른바 *깜깜이 심사*를 겪습니다. 이 프로젝트는 일본 [RetroHazard Immigration Dashboard](https://dashboard.retrohazard.jp)를 벤치마킹하여, 공개된 통계만으로 처리기간을 예측·시각화합니다.

## 주요 기능

- **처리 소요일 예측기** — 사무소·비자·신청월을 고르면 즉시 예상 소요일과 완료일(ETA), 95% 신뢰구간을 계산
- **관할구역 지도** — 14개 출입국사무소의 관할 시도를 색으로 표시하고, 관할 인구·면적·인구밀도와 예상 소요일을 함께 제공
- **처리건수 추이** — 실측 52개월(2022.01–2026.04) + OLS 예측 6개월을 하나의 차트로
- **비자별 분석** — 비자 종류별 예상 처리기간, 비중, 장기 트렌드
- **사무소별 순위·계절성** — 적체가 심한 사무소와 월별 성수기 파악
- 다크/라이트 테마(시스템 감지) · 한국어/영어 다국어 · 반응형(모바일/태블릿/데스크탑)

## 예측 모델

월별 처리건수를 **OLS 추세 + 계절성 분해 모델**로 학습했습니다.

| 지표 | 값 |
|---|---|
| 결정계수 (R²) | 0.958 |
| 평균절대오차율 (MAE) | 5.1% |
| RMSE | 16,048건 |
| 월 추세 | +2,802건/월 |

성수기는 8월(+44,948건), 비수기는 1월입니다. 사무소·비자별 소요일은 여기에 사무소 업무부하 가중치와 비자 처리 복잡도를 반영해 산출합니다.

## 데이터 수집 자동화

원본 통계월보는 robots.txt 차단과 세션 인증 때문에 자동 크롤링이 불가능합니다. 따라서 **"월 1회 파일 다운로드"라는 단 한 번의 수동 작업**만 사람이 하고, 이후 전 과정을 자동화했습니다.

- **GitHub Actions** (`deploy.yml`) — 데이터 파일이 갱신되면 파이썬 파이프라인을 실행해 CSV를 만들고 GitHub Pages로 자동 재배포
- **n8n 노코드 워크플로우** (`n8n_workflow.json`) — 매월 통계월보 발행일에 다운로드 알림 메일을 보내고, 폴더 감지로 파이프라인을 자동 트리거

## 구성

| 파일 | 설명 |
|---|---|
| `korea_immigration_dashboard.html` | 단일 파일 대시보드 (HTML/CSS/JS) |
| `immigration_data_pipeline.py` | 통계월보 → 정제 → OLS 예측 → CSV 파이프라인 |
| `immigration_realdata_full.csv` | 월별 실측·보간 데이터 (52개월) |
| `immigration_mock_1000.csv` | 사무소·비자별 모의 처리 데이터 |
| `deploy.yml` | GitHub Actions 자동 배포 워크플로우 |
| `n8n_workflow.json` | n8n 노코드 자동화 워크플로우 |

## 로컬 실행

대시보드는 별도 빌드 없이 `korea_immigration_dashboard.html`을 브라우저에서 열기만 하면 됩니다. (지도·차트 라이브러리는 CDN에서 로드)

파이프라인을 직접 돌리려면:

```bash
pip install pandas numpy openpyxl
python immigration_data_pipeline.py ./data/raw
```

## 웹 호스팅 (GitHub Pages)

1. 저장소 → **Settings → Pages**
2. Source를 **GitHub Actions**로 지정 (또는 간단히 `main` 브랜치 루트)
3. `korea_immigration_dashboard.html`을 `index.html`로 두면 주소가 깔끔해집니다

## 데이터 출처 및 면책

- 데이터: 법무부 출입국·외국인정책본부 **통계월보** (공공누리 제1유형)
- 관할 인구·면적: 행정안전부 주민등록인구, 행정구역 면적 (근사)
- **본 대시보드의 모든 수치는 통계 모델 기반 추정치이며 법적 효력이 없습니다.** 실제 처리기간은 개별 사안에 따라 다릅니다.
