# 출입국 행정 투명성 DX 대시보드

> 공공데이터(법무부 통계월보)를 가공해 **출입국 사무소·비자 종류별 비자 처리 예상기간**을 예측하고, 인터랙티브 대시보드로 공개하는 디지털 전환(DX) 프로젝트입니다.

한국에 체류하는 약 287만 외국인은 비자 변경·연장 신청 후 "언제 처리되는지" 알 수 없는 이른바 *깜깜이 심사*를 겪습니다. 이 프로젝트는 일본 [RetroHazard Immigration Dashboard](https://dashboard.retrohazard.jp)를 벤치마킹하여, 공개된 통계만으로 처리기간을 예측·시각화합니다.

## 주요 기능

- **처리 소요일 예측기** — 사무소·비자·신청월을 고르면 즉시 예상 소요일과 완료일(ETA), 95% 신뢰구간을 계산
- **관할구역 지도** — 14개 출입국사무소의 관할 시도를 색으로 표시하고, 관# 출입국 행정 투명성 대시보드

**한국어** · [English](README.en.md) · [日本語](README.ja.md)

법무부 출입국·외국인정책 통계를 가공해, 전국 17개 출입국·외국인관서의 **관할구역**과 비자 **처리 예상 소요일**을 단일 HTML 대시보드로 시각화하는 프로젝트입니다. 한국에 체류하는 외국인이 자기 지역의 담당 사무소, 서류 접수처(본청·출장소), 예상 처리기간을 한 화면에서 확인하도록 만드는 것이 목적입니다.

비자 연장·변경 신청 후의 처리기간은 공개되지 않아 신청자가 진행 상황을 알기 어렵습니다. 이 대시보드는 공개 통계만으로 그 기간을 추정하고, 관할·접수처 정보를 함께 제공합니다.

## 기능

- **처리 소요일 예측기** — 사무소·비자·신청월 입력 → 예상 소요일, 완료 예정일(ETA), 95% 신뢰구간
- **관할구역 지도(17개 사무소)** — 251개 시·군·구를 공식 관할에 따라 사무소 영역으로 병합하고 예상 소요일로 색상화. 사무소 호버 시 관할 시·군·구 + 서류 접수처(본청·출장소) + 처리 통계 표시. 지도가 어려운 이용자를 위한 격자형 **칸 보기** 제공
- **지역 유입·유출** — 시도별 월 순증감(신규 유입−유출) 코로플레스
- **처리건수 추이** — 실측 52개월(2022.01–2026.04) + OLS 6개월 예측
- **비자별 분석 · 비자 구성 · 계절성 · 사무소 순위**
- 한국어/영어/일본어 · 다크/라이트(시스템 감지) · 반응형(모바일/태블릿/데스크탑)

## 관할 매핑

17개 사무소 관할은 **출입국관리사무소 현황**(법무부 공식 관할구역)을 기준으로 산정합니다.

1. 「등록외국인 시군구별 거주 현황」(2026.04, 255행)에서 시·군·구별 등록외국인을 추출
2. 공식 관할표에 따라 각 시·군·구를 17개 사무소에 배정. 출장소는 모(母)사무소로 롤업하되, **교차 배정**은 공식대로 반영 — 예: 경북 경주→울산, 경남 김해·밀양·양산→부산, 강원 철원→양주, 경기 김포→인천, 경기 안양·광명→서울
3. 배정 후 **미매핑 0건 · 합계 일치**(1,623,256) 검증
4. 시·군·구 폴리곤을 사무소 단위로 dissolve → 메르카토르 투영 → 경량 SVG path로 사전 베이킹

이 절차로, 웹 조사에 의존하던 기존 매핑의 교차 관할 오류 약 10건을 공식 자료로 교정했습니다. 사무소 툴팁에는 본청·출장소별 접수 관할도 함께 표기해, 외국인이 서류를 어디서 접수하고 어느 사무소가 처리하는지 구분할 수 있게 했습니다.

## 예측 모델

월별 처리건수를 OLS 추세 + 계절성 분해로 학습했습니다. R² 0.958, MAE 5.1%, RMSE 16,048건, 추세 +2,802건/월, 성수기 8월(+44,948건).

사무소·비자별 예상 소요일:

```
소요일   = 기준일 × 부하지수^0.5
부하지수 = (관할 등록외국인 ÷ 용량등급) ÷ 전국 평균
용량등급 = 청 1.30 / 사무소 1.00 / 출장소 0.55   (정원 비공개로 등급 근사)
```

화면에서 실측(REAL)과 추정(EST)을 구분 표기합니다. 현재 모델의 가정은 사무소 용량등급 하나이며, 나머지 입력값(관할 등록외국인·관할구역·유입/유출)은 출처 있는 실측입니다. 정원·처리실적이 공개되면 이 가정도 실측으로 대체됩니다.

## 기술 구성

- 단일 HTML 파일. 빌드·서버 불필요, 브라우저에서 바로 실행
- 지도: d3/topojson/외부 fetch 없이 시·군·구 경계를 사무소 관할로 병합·투영해 SVG path로 내장(자립형). viewBox 단일 좌표계로 관할 지도와 유입·유출 지도를 공유
- 폰트: 한국어(나눔고딕)·일본어(Noto Sans JP)를 사용 글자만 서브셋 → woff2 → base64 임베드(합계 약 248KB). 폰트 CDN 의존 없음, 오프라인 동작. 라틴·숫자는 시스템 글꼴
- 외부 의존: 차트 라이브러리(Chart.js) 1개

## 참고 사항

- 체류외국인(약 287만)과 등록외국인(약 162만)은 다른 모집단입니다. 모든 수치에 인구 기준을 병기합니다.
- 미등록 외국인에 대한 차별을 유발할 수 있는 불법체류율 등은 포함하지 않습니다.
- 본 대시보드는 추정치 기반이며 법적 효력이 없습니다. 공식 통계가 아니고 법무부와 무관한 독립 학술 프로젝트입니다.

## 데이터 수집 자동화

원본 통계월보는 robots.txt 차단과 세션 인증 때문에 자동 크롤링이 불가능합니다. 따라서 **"월 1회 파일 다운로드"라는 단 한 번의 수동 작업**만 사람이 하고, 이후 전 과정을 자동화했습니다.

- **GitHub Actions** (`deploy.yml`) — 데이터 파일이 갱신되면 파이썬 파이프라인을 실행해 CSV를 만들고 GitHub Pages로 자동 재배포
- **n8n 노코드 워크플로우** (`n8n_workflow.json`) — 매월 통계월보 발행일에 다운로드 알림 메일을 보내고, 폴더 감지로 파이프라인을 자동 트리거
